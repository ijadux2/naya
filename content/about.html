About Naya

A modern systems programming language with Zig-like power and bash-like simplicity

Philosophy

Naya was designed with a clear philosophy: combine the simplicity of bash with the power of modern systems languages. We believe systems programming should be accessible without sacrificing performance or control.

  • Simple Syntax
    Clean, readable syntax inspired by modern languages
  • Zero-Cost Abstractions
    Compile-time features with no runtime overhead
  • Memory Safety
    Manual memory management with safety features

Design Goals

Naya achieves its goals through careful language design and tooling integration:

  • C ABI Compatibility
    Seamless integration with existing C libraries
  • Professional Tooling
    LSP support, build system, package manager
  • Cross-Platform
    Linux, macOS, and Windows support

Language Features

Type System

  • Static typing with inference
  • Generic programming
  • Union and optional types
  • Compile-time evaluation

Memory Management

  • Safe and unsafe pointers
  • Arena allocation
  • Defer statements
  • Automatic cleanup

Error Handling

  • Result types
  • Try-catch expressions
  • Pattern matching
  • Compile-time checks
